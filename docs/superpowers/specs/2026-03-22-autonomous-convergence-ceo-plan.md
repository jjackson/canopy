## Vision

### 10x Check
The 10x version of Canopy isn't "watches sessions and proposes tool improvements."
It's **an autonomous product manager for the entire AI development ecosystem.**
It watches every session, applies CEO-level thinking from gstack's cognitive
patterns, invokes any skill from any plugin system to validate proposals, spins
up parallel implementations in worktrees, and produces a strategic brief that
reads like a VP of Engineering's Monday morning report.

### Platonic Ideal
A brilliant, tireless CTO who never sleeps. You wake up to: "While you slept,
I analyzed your 4 sessions. Two patterns emerged: your funder research workflow
has a 3-step bottleneck I've now automated, and the Salesforce MCP drops sessions
every ~2 hours which I've fixed with auto-refresh. I also noticed you're building
CommCare apps from scratch each time — I've created a template system. All changes
are on feature branches with passing tests. Here's what I'm planning for tomorrow."

## Key Architectural Decision

**Skill Runner via slash command invocation.** The autonomous loop invokes any
skill from any plugin system (gstack, superpowers, canopy) by sending slash
commands to `claude -p`. Plugin-agnostic, zero coupling. Interactive skills
receive an instruction to auto-select the recommended option.

## Execution Platform

**Claude Code Agent Teams** (experimental, v2.1.32+) replaces custom subprocess
orchestration and emdash dependency for the autonomous loop:
- Pipeline IS the team lead
- Each proposal implementation IS a teammate
- Shared task list IS the run log
- Teammates work in isolated worktrees
- Native coordination, messaging, and task dependencies (DAGs)

Emdash remains useful for Jonathan's interactive work but is NOT required for
Canopy's autonomous loop.

## Patterns Adopted

**From Citadel:**
- Circuit breaker: 3 consecutive failures → stop, log, try different approach
- Tiered routing: classify by complexity, route to cheapest tier (skill/marshal/fleet)
- Campaign persistence: multi-day arcs in markdown files
- Discovery relay: ~500-token briefs shared between parallel agents

**From Super-Ralph:**
- Dual-condition exit gate: require "tests pass" AND "explicit completion marker"
- Evidence-based verification: capture literal test output, not just exit codes
- Rate limiting: configurable calls/hour cap

**From Gstack:**
- CEO cognitive patterns in analyzer/proposer prompts
- Completeness Principle for proposal evaluation
- /review, /qa, /plan-eng-review invocable by autonomous loop

## Critical Design Decisions
- Slash command invocation for skill runner (not SKILL.md injection)
- Auto-select recommended for all interactive decisions in headless mode
- Claude Code Agent Teams for parallel execution (not emdash, not custom subprocess)
- Scanner replaces session-log for transcript discovery
- Verification plans determine proposal priority
- Self-owned repos: auto-merge. Team repos: PR. External: registry-only.
- Max 3 proposals per cycle. Per-session budget cap + rate limiting.
- Circuit breaker prevents runaway API spend
