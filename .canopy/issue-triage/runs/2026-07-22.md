### Feasibility note (found during triage)

The fix is feasible **without** a service-account/MCP dependency: `gog` v0.12
already exposes in-place Docs atoms — `gog docs clear <docId>` + `gog docs write
<docId> --file <f>` (write replaces the body by default), keeping the same doc
ID/URL.

**Wrinkle (needs live validation):** `gog docs write` is **plain-text** — no
`--markdown`/`--html` flag — so a naive `clear` + `write --file <md>` would drop
the heading/bold/list formatting the create path gets from HTML→Doc conversion.
The markdown-rendering in-place path must be confirmed against a live test Doc
before shipping. This is the Phase 4.5 "unvalidated" exception — build behind a
live check, do not blind-ship.
